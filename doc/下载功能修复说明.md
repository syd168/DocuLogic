# 下载功能 Bug 修复说明

## 问题描述

在 Docker + Nginx 反向代理环境下，"完整结果（ZIP，含图片）"下载功能无法使用。

## 根本原因

1. **ZIP 文件生成条件限制**：`{job_id}_result.zip` 只在 `image_output_mode == "separate"` 且有 assets 文件夹时才会生成
2. **前端始终显示下载链接**：无论是否生成了 ZIP，前端都会显示下载链接
3. **后端返回 404**：当用户点击下载时，如果 ZIP 文件不存在，后端会返回 404 错误

## 修复内容

### 1. 后端代码修改 (`web/app/model_inference.py`)

**修改点 1：单页处理函数 `_process_single_image`**
- **原逻辑**：仅在有 assets 时才生成 ZIP
- **新逻辑**：始终生成 ZIP 文件，包含基础文件（markdown、raw、visualization），如果有 assets 则额外添加

**修改点 2：PDF 多页处理函数 `_process_pdf`**
- **原逻辑**：仅在有 assets 时才生成 ZIP
- **新逻辑**：始终生成 ZIP 文件，包含基础文件和可视化结果，如果有 assets 则额外添加

**优势**：
- 确保每次解析完成后都有 ZIP 文件可供下载
- ZIP 文件始终包含核心输出（markdown、raw、visualization）
- 在 separate 模式下还会包含提取的图片资源

### 2. 下载接口优化 (`web/app/main.py`)

为所有下载接口添加了明确的响应头：
```python
headers={
    "Content-Disposition": f'attachment; filename="{filename}"',
    "Cache-Control": "no-cache",
}
```

**作用**：
- `Content-Disposition: attachment` - 强制浏览器下载而非预览
- `Cache-Control: no-cache` - 避免缓存导致的问题

### 3. Nginx 配置优化 (`docker/nginx/default.conf`)

优化了 `/download/` 路径的配置：
```nginx
# 大文件下载支持
proxy_buffering off;
proxy_request_buffering off;
proxy_cache off;

# 超时设置（大文件可能需要更长时间）
proxy_connect_timeout 600s;
proxy_send_timeout 600s;
proxy_read_timeout 600s;

# 禁用 gzip，避免压缩大文件导致额外开销
gzip off;
```

**改进**：
- 禁用缓冲和缓存，实现流式传输
- 超时时间从 300s 增加到 600s，支持更大的文件
- 禁用 gzip 压缩，减少 CPU 开销

## 技术细节

### ZIP 文件内容结构

**基础模式（base64）**：
```
{job_id}_result.zip
├── {job_id}.mmd              # 转换后的 markdown
├── {job_id}_raw.mmd          # 原始输出
└── {job_id}_vis.png          # 可视化结果（单页）或 ZIP（多页）
```

**分离模式（separate）**：
```
{job_id}_result.zip
├── {job_id}.mmd              # 转换后的 markdown
├── {job_id}_raw.mmd          # 原始输出
├── {job_id}_vis.png          # 可视化结果
└── assets/                   # 提取的图片资源
    ├── image_001.png
    ├── image_002.png
    └── ...
```

## 注意事项

1. **磁盘空间**：由于始终生成 ZIP 文件，请确保有足够的磁盘空间
2. **清理策略**：建议定期清理旧的解析结果以释放空间
3. **性能影响**：生成 ZIP 会增加少量处理时间（通常 < 1 秒）
