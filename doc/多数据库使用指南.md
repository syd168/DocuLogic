# 多数据库支持使用指南

本文档介绍 DocuLogic 的多数据库支持功能，包括 SQLite、MySQL、MariaDB 和 PostgreSQL。

## 📊 支持的数据库类型

| 数据库 | 版本要求 | 适用场景 | Docker Profile |
|--------|---------|---------|---------------|
| **SQLite** | 3.x+ | 开发环境、小规模部署 | 无（默认） |
| **MySQL** | 8.0+ | 生产环境、中等规模 | `mysql` |
| **MariaDB** | 10.5+ | 生产环境、MySQL 替代品 | `mariadb` |
| **PostgreSQL** | 16+ | 生产环境、大规模部署 | `postgresql` |

---

## 🚀 快速开始

### 方式一：Docker 部署（推荐）

#### 1. 选择数据库类型

编辑 `.env` 文件，修改 `DATABASE_TYPE`：

```bash
# SQLite（默认，无需额外配置）
DATABASE_TYPE=sqlite

# MySQL
DATABASE_TYPE=mysql

# MariaDB
DATABASE_TYPE=mariadb

# PostgreSQL
DATABASE_TYPE=postgresql
```

#### 2. 一键部署

```bash
cd docker
./deploy.sh
```

脚本会自动：
- ✅ 检测数据库类型
- ✅ 启动对应的数据库容器（如需要）
- ✅ 自动导入初始化 SQL 文件
- ✅ 创建默认管理员账号（admin / admin123）

#### 3. 验证部署

```bash
# 查看服务状态
docker compose ps

# 查看日志
docker compose logs -f doculogic-mysql    # 或 mariadb/postgresql
```

---

### 方式二：本地部署

#### 前置要求

- Python 3.9+
- 目标数据库已安装并运行
- 已创建数据库和用户

#### 步骤

##### 1. 安装依赖

```bash
pip install -r requirements.txt
```

根据选择的数据库，可能需要额外安装：

```bash
# MySQL/MariaDB
pip install pymysql

# PostgreSQL
pip install psycopg2-binary
```

##### 2. 配置 .env

```bash
cp .env.example .env
```

编辑 `.env`：

```ini
# 选择数据库类型
DATABASE_TYPE=postgresql

# PostgreSQL 配置
PG_HOST=localhost
PG_PORT=5432
PG_USER=doculogic
PG_PASSWORD=your_password
PG_DATABASE=doculogic
```

##### 3. 初始化数据库

###### MySQL/MariaDB

```bash
# 方法1：使用命令行工具
mysql -u root -p < web/data/mysql.sql

# 方法2：使用迁移脚本导出 SQLite 数据
python3 migrate_sqlite_to_mysql.py export
```

###### PostgreSQL

```bash
# 使用 psql 工具
psql -U postgres -d doculogic -f web/data/postgresql.sql
```

##### 4. 启动应用

```bash
python3 web_server.py
```

## 📁 数据库文件说明

### 初始化 SQL 文件位置

```
web/data/
├── app.db              # SQLite 数据库文件（自动生成）
├── mysql.sql           # MySQL 初始化脚本
├── mariadb.sql         # MariaDB 初始化脚本
└── postgresql.sql      # PostgreSQL 初始化脚本
```

### 自动导入机制

**Docker 部署时**，数据库容器首次启动会自动执行 `/docker-entrypoint-initdb.d/init.sql`：

- MySQL: `web/data/mysql.sql` → 容器内 `/docker-entrypoint-initdb.d/init.sql`
- MariaDB: `web/data/mariadb.sql` → 容器内 `/docker-entrypoint-initdb.d/init.sql`
- PostgreSQL: `web/data/postgresql.sql` → 容器内 `/docker-entrypoint-initdb.d/init.sql`

**注意**：仅在数据目录为空时执行（即首次启动）。

## 🔧 数据库切换

### 从 SQLite 切换到 MySQL

1. 停止服务：
   ```bash
   cd docker && docker compose down
   ```

2. 修改 `.env`：
   ```ini
   DATABASE_TYPE=mysql
   MYSQL_PASSWORD=your_secure_password
   ```

3. （可选）导出 SQLite 数据：
   ```bash
   python3 migrate_sqlite_to_mysql.py export
   ```

4. 重新启动：
   ```bash
   cd docker && ./deploy.sh
   ```

### 从 MySQL 切换到 PostgreSQL

1. 停止服务：
   ```bash
   cd docker && docker compose down --volumes  # 删除 MySQL 数据
   ```

2. 修改 `.env`：
   ```ini
   DATABASE_TYPE=postgresql
   PG_PASSWORD=your_secure_password
   ```

3. 重新启动：
   ```bash
   cd docker && ./deploy.sh
   ```

## 💾 数据备份与恢复

### MySQL

```bash
# 备份
docker exec doculogic-mysql mysqldump -uroot -p doculogic > backup_$(date +%Y%m%d).sql

# 恢复
docker exec -i doculogic-mysql mysql -uroot -p doculogic < backup_20260408.sql
```

### MariaDB

```bash
# 备份
docker exec doculogic-mariadb mariadb-dump -uroot -p doculogic > backup_$(date +%Y%m%d).sql

# 恢复
docker exec -i doculogic-mariadb mariadb -uroot -p doculogic < backup_20260408.sql
```

### PostgreSQL

```bash
# 备份
docker exec doculogic-postgresql pg_dump -U postgres doculogic > backup_$(date +%Y%m%d).sql

# 恢复
docker exec -i doculogic-postgresql psql -U postgres -d doculogic < backup_20260408.sql
```

### SQLite

```bash
# 直接复制数据库文件
cp web/data/app.db backup_$(date +%Y%m%d).db
```

## 🐛 常见问题

### Q1: 数据库容器无法启动？

检查数据目录权限：

```bash
# 确保数据目录存在且可写
mkdir -p ~/doculogic/data/mysql
chmod 755 ~/doculogic/data/mysql
```

### Q2: 初始化 SQL 未执行？

仅当数据目录为空时才执行。如需重新初始化：

```bash
# 警告：这将删除所有数据！
rm -rf ~/doculogic/data/mysql/*
docker compose up -d mysql
```

### Q3: 如何查看数据库连接状态？

```bash
# 查看应用日志
docker compose logs -f doculogic | grep -i database

# 进入数据库容器
docker exec -it doculogic-mysql mysql -uroot -p
docker exec -it doculogic-postgresql psql -U postgres
```

### Q4: 性能优化建议

**MySQL/MariaDB**:
- 增加 `innodb_buffer_pool_size`
- 调整 `max_connections`

**PostgreSQL**:
- 增加 `shared_buffers`
- 调整 `work_mem`
- 启用 `pg_stat_statements` 监控
