### #1. SMS HTTP URL 未校验协议 (High) 🔴

**位置**: `web/app/sms_svc.py:42-75`

**问题描述**:
```python
url = (cfg.get("http_url") or "").strip()
# ❌ 未校验 URL 协议
req = urllib.request.Request(url, data=payload, method="POST", headers=headers)
```

管理员可配置任意 URL，包括：
- `file:///etc/passwd` - 读取本地文件
- `http://169.254.169.254/latest/meta-data/` - AWS 元数据泄露
- `http://internal-service:8080/admin` - 内网服务访问

**影响**:
- SSRF（服务器端请求伪造）攻击
- 内网信息泄露
- 云服务商元数据窃取

**利用场景**:
1. 攻击者获取管理员权限
2. 在系统设置中配置恶意 SMS URL
3. 触发短信发送时，服务器发起恶意请求
4. 窃取内网敏感信息或云凭证

**修复建议**:
```python
from urllib.parse import urlparse
import ipaddress
import socket

def validate_sms_url(url: str) -> bool:
    """校验 SMS HTTP URL 的安全性"""
    parsed = urlparse(url)
    
    # 1. 仅允许 http/https 协议
    if parsed.scheme not in ("http", "https"):
        raise ValueError(f"不支持的协议: {parsed.scheme}，仅允许 http/https")
    
    # 2. 禁止内网地址（防止 SSRF）
    hostname = parsed.hostname
    if not hostname:
        raise ValueError("无效的 URL")
    
    try:
        # 解析域名
        ip_addresses = socket.getaddrinfo(hostname, None)
        for addr in ip_addresses:
            ip = ipaddress.ip_address(addr[4][0])
            if ip.is_private or ip.is_loopback or ip.is_link_local:
                raise ValueError(f"禁止访问内网地址: {ip}")
    except socket.gaierror:
        raise ValueError(f"无法解析域名: {hostname}")
    
    return True

# 在 send_sms_code 中使用
url = (cfg.get("http_url") or "").strip()
if not url:
    raise RuntimeError("短信 HTTP 回调地址未配置")

validate_sms_url(url)  # ✅ 添加校验
```

**优先级**: High  
**修复时间**: 2小时  
**测试建议**: 尝试配置 `file://`、`gopher://`、内网 IP 等恶意 URL

### #2. 密码重置后旧 Token 仍有效 (High) 🔴

**位置**: `web/app/routers/auth.py:713-719`

**问题描述**:
```python
u.hashed_password = hash_password(body.new_password)
db.commit()
# ❌ 未销毁用户的活跃会话
```

虽然已有 `session_manager.destroy_user_session()` 函数，但密码重置接口未调用。

**影响**:
- 密码重置无法立即阻止未授权访问
- 被盗设备仍可访问账户
- 违反安全最佳实践（OWASP ASVS 2.4.6）

**利用场景**:
1. 攻击者盗取用户设备或 Token
2. 用户发现异常并重置密码
3. 攻击者的 Token 仍然有效，可继续访问
4. 直到 Token 自然过期（默认7天）

**修复建议**:
```python
from ..session_manager import destroy_user_session

u.hashed_password = hash_password(body.new_password)
db.commit()

# ✅ 销毁所有活跃会话
destroy_user_session(u.id, reason="password_reset")
_log.info(f"用户 {u.id} 密码已重置，所有会话已销毁")

return {"message": "密码重置成功，请使用新密码登录"}
```

**优先级**: High  
**修复时间**: 30分钟  
**测试建议**: 重置密码后立即使用旧 Token 访问 API

### #3. JWT_SECRET 警告未阻止服务启动 (Medium) 🟡

**位置**: `web/app/auth_security.py:26-35`

**问题描述**:
```python
if JWT_SECRET == "change-me-use-openssl-rand-hex-32":
    import warnings
    warnings.warn(...)  # ❌ 仅警告，不阻止启动
```

当前实现仅发出 `RuntimeWarning`，但服务仍可启动。生产环境中：
- 管理员可能忽略警告
- 容器重启后仍使用默认密钥
- Docker 日志可能被覆盖，警告丢失

**影响**:
- 高风险配置仍可生效
- 违背"Fail Securely"原则
- 所有 JWT Token 可被预测和伪造

**修复建议**:
```python
if JWT_SECRET == "change-me-use-openssl-rand-hex-32":
    # 生产环境强制退出
    if os.environ.get("ENVIRONMENT", "").lower() in ("production", "prod"):
        raise RuntimeError(
            "⛔ 生产环境检测到默认 JWT_SECRET！\n"
            "请立即修改 .env 文件中的 JWT_SECRET 为强随机值。\n"
            "生成命令：python3 -c \"import secrets; print(secrets.token_hex(32))\""
        )
    
    # 开发环境仅警告
    import warnings
    warnings.warn(...)
```

**优先级**: Medium → High（生产环境）  
**修复时间**: 30分钟  
**测试建议**: 设置 `ENVIRONMENT=production` 并使用默认密钥启动

### #4. SMS HTTP Headers JSON 注入风险 (Medium) 🟡

**位置**: `web/app/sms_svc.py:56-58`

**问题描述**:
```python
extra = cfg.get("http_headers") or {}
if isinstance(extra, dict):
    headers.update({k: str(v) for k, v in extra.items() if v is not None})
```

虽然从数据库读取，但管理员可通过 API 设置恶意头部：
- `Authorization: Bearer <token>` - 劫持其他服务
- `Cookie: session=xxx` - 会话固定攻击
- 自定义头部导致下游服务混淆

**影响**:
- HTTP 头部注入
- 下游服务认证绕过
- CRLF 注入（如果值包含换行符）

**修复建议**:
```python
# 白名单允许的自定义头部
ALLOWED_CUSTOM_HEADERS = {
    "x-api-key",
    "x-auth-token",
    "x-client-id",
    "x-request-id",
}

extra = cfg.get("http_headers") or {}
if isinstance(extra, dict):
    for k, v in extra.items():
        k_lower = k.lower().strip()
        # 1. 禁止覆盖关键头部
        if k_lower in ("content-type", "user-agent", "host", "authorization", "cookie"):
            _log.warning(f"禁止覆盖关键头部: {k}")
            continue
        
        # 2. 仅允许白名单头部
        if k_lower not in ALLOWED_CUSTOM_HEADERS and not k_lower.startswith("x-"):
            _log.warning(f"不允许的自定义头部: {k}")
            continue
        
        # 3. 清理头部值（防止 CRLF 注入）
        v_clean = str(v).replace("\r", "").replace("\n", "")
        headers[k] = v_clean
```

**优先级**: Medium  
**修复时间**: 1小时  
**测试建议**: 尝试设置 `Authorization`、`Cookie` 等关键头部

### #6. SQL LIKE 通配符未转义 (Low) 🟢

**位置**: `web/app/main.py:617-619`

**问题描述**:
```python
if filename and filename.strip():
    fn = f"%{filename.strip()}%"
    q = q.filter(ParseJob.original_filename.ilike(fn))
```

用户输入 `_` 或 `%` 会被解释为 SQL 通配符：
- `_` 匹配任意单字符
- `%` 匹配任意多字符

**影响**:
- 绕过文件名过滤
- 枚举其他用户的任务
- 信息泄露

**修复建议**:
```python
if filename and filename.strip():
    # 转义特殊字符
    escaped = filename.strip().replace("\\", "\\\\").replace("%", "\\%").replace("_", "\\_")
    fn = f"%{escaped}%"
    q = q.filter(ParseJob.original_filename.ilike(fn, escape="\\"))
```

**优先级**: Low  
**修复时间**: 30分钟  
**测试建议**: 搜索包含 `%` 或 `_` 的文件名

### #8. Docker 健康检查过于简单 (Info) ℹ️

**位置**: `docker/Dockerfile:74-75`

**问题描述**:
```dockerfile
HEALTHCHECK --interval=30s --timeout=10s --start-period=60s --retries=3 \
    CMD curl -f http://localhost/health || exit 1
```

仅检查 Nginx 是否响应，未检查：
- 后端 FastAPI 服务是否正常
- 数据库连接是否健康
- 模型是否加载成功

**修复建议**:
```dockerfile
HEALTHCHECK --interval=30s --timeout=10s --start-period=60s --retries=3 \
    CMD curl -f http://localhost/health && \
        curl -f http://localhost:8000/health && \
        python -c "import sys; sys.exit(0 if check_db_health() else 1)" || exit 1
```

或在应用层实现综合健康检查：
```python
@app.get("/health")
async def health_check(db: Session = Depends(get_db)):
    checks = {
        "nginx": "ok",
        "backend": "ok",
        "database": check_db_health(db),
        "model": model is not None,
    }
    
    if all(checks.values()):
        return {"status": "healthy", "checks": checks}
    else:
        raise HTTPException(status_code=503, detail={"status": "unhealthy", "checks": checks})
```

**优先级**: Info  
**修复时间**: 1小时  
**测试建议**: 停止数据库或模型服务，观察健康检查状态
