# 手动卸载模型功能说明

## 使用流程

```
管理员登录
    ↓
进入「系统设置」→「模型管理」
    ↓
查看模型状态（已加载）
    ↓
点击「🗑️ 卸载模型」按钮
    ↓
确认对话框：「确定要卸载模型吗？」
    ↓
确认后执行卸载
    ↓
显示成功消息 + 更新状态
    ↓
显存已释放 ✅
```

## 用户提示

### 成功消息
```
✅ 模型已成功卸载

显存已释放

下次上传文档时将自动重新加载模型
```

### 警告消息
- 下载任务进行中：`下载任务正在进行中，请等待下载完成后再尝试卸载`
- 模型未加载：`ℹ️ 模型未加载，无需卸载`

### 确认对话框
```
确定要卸载模型吗？这将释放显存，下次上传文档时将自动重新加载。

[取消] [卸载]
```

## 技术优势

1. **按需释放**: 用户可以在不需要解析文档时主动释放显存
2. **自动重载**: 下次上传文档时会自动重新加载模型（懒加载机制）
3. **安全防护**: 
   - 下载任务进行时禁止卸载
   - 需要二次确认防止误操作
   - 仅管理员可操作
4. **友好体验**:
   - 清晰的提示信息
   - 加载状态动画
   - 实时状态更新

## 与其他功能的配合

### 与懒加载机制配合

| 场景 | 行为 |
|------|------|
| 启动服务 | 不加载模型（显存占用极小） |
| 首次上传 | 自动加载模型 |
| 手动卸载 | 立即释放显存 |
| 再次上传 | 自动重新加载 |
| 空闲5分钟 | 自动卸载（超时机制） |

### 与模型重载对比

| 功能 | 重新加载模型 | 卸载模型 |
|------|-------------|---------|
| 作用 | 先卸载再加载 | 仅卸载 |
| 使用场景 | 配置更改后、加载失败重试 | 释放显存、暂时不用 |
| 显存变化 | 不变 | 释放 |
| 后续操作 | 立即可用 | 需重新加载 |

## 注意事项

1. **权限要求**: 仅管理员可见和操作
2. **状态依赖**: 仅在模型已加载时显示按钮
3. **并发控制**: 下载任务进行时禁止卸载
4. **自动重载**: 卸载后下次上传会自动加载，无需担心
